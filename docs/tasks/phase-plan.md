# Delivery Robot 阶段计划

## 阶段 1：纯 Rust 基础期
目标：不依赖 ROS2，先完成核心逻辑。

范围：
- 栅格地图
- A* 路径规划
- 任务状态机
- 配置加载
- 单元测试
- 简单 CLI 演示

完成标准：
- 能加载一个简单地图
- 能从起点规划到终点
- 能表示任务状态流转
- 核心模块都有单元测试
- CLI 能打印规划结果

## 阶段 2：ROS2 入门期
目标：把 Rust 核心模块包装成 ROS2 包。

范围：
- robot_msgs
- map_server
- planner_node
- 参数系统
- launch 文件

完成标准：
- 能通过 ROS2 调用规划器
- 能通过 ROS2 获取地图数据
- 自定义消息/服务可以成功构建

## 阶段 3：仿真闭环期
目标：在仿真环境中跑通最小配送闭环。

范围：
- 仿真适配层
- 定位输入
- 控制器节点
- 障碍物输入
- 导航闭环

完成标准：
- 机器人能在仿真中移动到目标点

## 开发规则
- 一次只推进一个阶段
- 每个模块都必须有测试
- 每个阶段结束都要有一个可运行演示
- 每个阶段结束都要补学习记录
